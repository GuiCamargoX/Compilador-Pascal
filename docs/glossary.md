# Compiler Glossary for Beginners 📖

Use this file when a term sounds unfamiliar.

## Source code

The input program written by a user (here: Pascal-like `.pas` files).

## Token

A categorized piece of source text (for example: keyword, identifier, literal, operator).

In this repo: represented by `Token` in `Compiler/FrontEnd/Lexer/Token.cs`.

## Lexeme

The exact text from source that produced a token.

Example:

- source text: `while`
- token type: `TK_WHILE`
- lexeme: `while`

## Lexer / Scanner

Compiler stage that reads characters and emits tokens.

In this repo: `Compiler/FrontEnd/Lexer/Scanner.cs`.

## Parser

Compiler stage that checks token order against grammar rules.

In this repo: `Compiler/FrontEnd/Parser/Parser*.cs`.

## Grammar rule

A definition of valid token sequences.

Example startup rule in this project:

- `program <id> ( <id> [, <id>]* );`

## Semantic analysis

Checks meaning, not just shape.

Examples:

- variable must be declared before use
- scope must be respected

## Symbol table

Data structure that stores identifier information (name, kind, type, scope, address).

In this repo: `Compiler/FrontEnd/Semantics/SymbolTable.cs`.

## Scope

Region where a declaration is visible.

In this repo, scopes are pushed/popped with `OpenScope()` and `CloseScope()`.

## Code generation

Stage that emits target instructions from parsed/validated structures.

In this repo: parser calls `GenerateMepa(...)`.

## Intermediate representation (IR)

A lower-level format generated by compiler stages before final machine code.

In this project, MEPA-like instructions play that role.

## MEPA

The instruction format generated by this compiler into `Mepa.txt`.

Example instructions:

- `INPP` (program start)
- `AMEM` (allocate memory)
- `DSVF` (conditional jump)
